# UML
객체 간의 관계를 나타내는 구조 다이어그램과 시간 흐름에 따라 객체들이 어떻게 조합하여 기능을 구현하는지 나타내는 동작 다이어그램이 있다.

# 클래스 다이어그램
대표적인 구조 다이어그램.
클래스란, 동일한 속성을 가지고 동일한 행동을 수행하는 객체의 집합.
클래스의 구조를 살펴보면 클래스의 이름, 필드, 메서드로 구성된다.
따라서 상단부에 이름, 중간부에 필드, 하단부에 메서드를 표기한 것이 클래스 다이어그램이다.

# 인터페이스
인터페이스란 책임이다. 해당 객체가 반드시 가지고 있거나, 수행해야할 것들을 명시하는 곳이 인터페이스이다.
자바에서는 시스템 구조 상, 하나의 상위 클래스만 존재한다. 그러니까 부모는 하나밖에 존재할 수 없다.
인터페이스를 이용하면 자바의 이러한 구조를 피하여, 다중 상속이 가능하다.
